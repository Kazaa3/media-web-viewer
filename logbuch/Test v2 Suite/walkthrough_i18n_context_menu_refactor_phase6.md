# Walkthrough: I18n Context Menu Refactor (Phase 6)

## Ziel
- Vollständige I18n-Abdeckung und SVG-basierte UI für alle technischen Kontextmenüs.

---

## Umsetzungsschritte
1. **i18n.json Dictionary:**
   - Erweiterung um alle Kontextmenü-Keys (z.B. Kopieren, Löschen, Umbenennen, Details, etc.).
2. **HTML-Refactoring:**
   - Migration der Kontextmenü-Elemente in web/app.html auf SVG-Symbole und dynamische I18n-Tags.
3. **Performance & Konsistenz:**
   - SVG-Designsystem für schnelle, konsistente UI-Renderings implementiert.

---

## Review & Tests
- Überprüfung der neuen Kontextmenüs auf vollständige Übersetzbarkeit und UI-Performance.
- Sicherstellung, dass alle Menüs dynamisch und sprachabhängig gerendert werden.
