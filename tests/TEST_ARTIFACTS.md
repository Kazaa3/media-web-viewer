# Test Artifacts Documentation

This document describes the structure and management of files generated during test execution. 

## Central Artifact Storage: `tests/artifacts/`

All temporary files, logs, and reports generated by tests must be stored in `tests/artifacts/`. This directory is excluded from version control via `.gitignore`.

### Directory Structure

| Directory | Purpose | Retention |
| :--- | :--- | :--- |
| `tests/artifacts/logs/` | Log files from app startup, test runs, and CLI tools. | Temporary |
| `tests/artifacts/reports/` | JSON and XML reports (e.g., coverage, benchmark results). | Persistent (Local) |
| `tests/artifacts/screenshots/` | UI screenshots for failure analysis and comparison. | Temporary / Local |
| `tests/artifacts/data/` | Dynamic test data, media dummies, and temporary DBs. | Temporary |

## Git Ignore Policy

The following file types and directories are explicitly ignored in the root and `tests/` directory:

- `*.log`: All log files.
- `*.json` (inside `tests/`): All test reporting fragments.
- `*.txt` (inside `tests/`): Temporary text output.
- `tests/artifacts/`: The entire centralized storage.
- `tests/assets/`: Legacy asset storage (mirroring artifacts).

## Standardized Paths in Tests

When writing new tests or updating existing ones, use the following patterns for output:

```python
from pathlib import Path

# Base directory for artifacts
ARTIFACT_ROOT = Path(__file__).parents[2] / "artifacts"
LOG_DIR = ARTIFACT_ROOT / "logs"
REPORT_DIR = ARTIFACT_ROOT / "reports"

# Example: Writing a report
report_path = REPORT_DIR / "test_results.json"
```

---
*Note: This structure ensures a clean repository and easy cleanup of test-related debris.*
