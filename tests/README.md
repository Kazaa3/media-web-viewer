# Media Web Viewer - Unified Diagnostic Suite

This directory contains the modernized, modular testing infrastructure for the Media Web Viewer. The suite is designed for 100% automated verification of system health, database integrity, and media processing logic.

## 🚀 Quick Start
Run the complete diagnostic suite (**80+ stages**):
```bash
python3 tests/run_all.py
```

## 🏗️ Architecture
The testing framework is divided into **10 specialized Diagnostic Engines**:

1. **Ultimate Suite** (`tests/engines/suite_ultimate.py`): E2E integration, concurrency, and stress (29 stages).
2. **Items Suite** (`tests/engines/suite_items.py`): Metadata parsing, transcode logic, and ffprobe resolution (11 stages).
3. **Database Suite** (`tests/engines/suite_database.py`): Schema integrity and ACID compliance (4 stages).
4. **UI Suite** (`tests/engines/suite_ui.py`): Structural HTML integrity and CSS theme audits (3 stages).
5. **Env Suite** (`tests/engines/suite_env.py`): Dependency audits and deployment asset verification (3 stages).
6. **Player Suite** (`tests/engines/suite_player.py`): Playback lifecycle and HW acceleration (5 stages).
7. **Integrity Suite** (`tests/engines/suite_media_integrity.py`): Live media parsing and artwork extraction (5 stages).
8. **Network Suite** (`tests/engines/suite_network.py`): Live Eel/Bottle server and static file serving (5 stages).
9. **Quality Suite** (`tests/engines/suite_quality.py`): Eel API alignment and HTML sanity audits (5 stages).
10. **Automation Suite** (`tests/engines/suite_automation.py`): PyAutoGUI interaction and Selenium smoke tests (6 stages).
