# Media Web Viewer - Unified Diagnostic Suite

This directory contains the modernized, modular testing infrastructure for the Media Web Viewer. The suite is designed for 100% automated verification of system health, database integrity, and media processing logic.

## 🚀 Quick Start
Run the complete diagnostic suite (50+ stages):
```bash
python3 tests/run_all.py
```

## 🏗️ Architecture
The testing framework is divided into five specialized **Diagnostic Engines**:

1. **Ultimate Suite** (`tests/engines/suite_ultimate.py`):
   - **Coverage**: End-to-end integration, concurrency, stress, and fallback logic.
   - **Key Levels**: DB Concurrency (L27), Corrupt Media Recovery (L28), Transcoding Stress (L29).
2. **Items Suite** (`tests/engines/suite_items.py`):
   - **Coverage**: Metadata parsing, multi-track resolution, and data model mocking.
   - **Key Levels**: ALAC/WMA Transcode Logic (L08), Multi-Track ffprobe Probing (L11).
3. **Database Suite** (`tests/engines/suite_database.py`):
   - **Coverage**: Schema integrity, CRUD persistence, and transaction rollbacks.
4. **UI Suite** (`tests/engines/suite_ui.py`):
   - **Coverage**: Structural HTML integrity and CSS theme variable audits.
5. **Env Suite** (`tests/engines/suite_env.py`):
   - **Coverage**: Multi-dependency audits (9 core pkgs) and deployment asset verification.
6. **Player Suite** (`tests/engines/suite_player.py`):
   - **Coverage**: Seeking accuracy, playback session lifecycle, and hardware acceleration integration.
   - **Key Levels**: Seeking Injection (L01), Session Termination (L02), Float-Based Precision Seeking (L05).
